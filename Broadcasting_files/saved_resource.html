                                    Muaz Khan Owner John Carre • 10 months ago

                                    Actually this demo is trying to support all 3rd party signaling implementations e.g pubnub, pusher, firebase, websocket, socketio etc.

                                    This page explains how to use single socket per page: https://github.com/muaz-khan/WebRTC-Experiment/blob/master/Signaling.md

                                    I'll recommend this demo with similar functionality: https://rtcmulticonnection.herokuapp.com/demos/Video-Broadcasting.html

                                    mkdir demo
                                    cd demo
                                    git clone https://github.com/muaz-khan/RTCMultiConnection.git ./
                                    npm install
                                    node server

                                    Now open this URL:

                                    http://localhost:9001/demos/Video-Broadcasting.html

                                    Advantages:
                                    1) It is running & using your own custom socket.io server
                                    2) Only single socket connection is used per page
                                    3) It supports edge/safari11/firefox/opera/chrome/android/ios
                                    4) For android/ios, it supports both cordova and ionic

                                                                                                              Muaz Khan Owner Siavash Goudarzi • 4 years ago

                                                                                                              Both apps maybe using WebRTC and can be developed using WebRTC however media-stack (server) need to understand how to transcode/mux multiple streams or how to change sources etc. For playback or recording the streams (similar to periscope), media stack need to decode/encode and write down the RTP packets in a specific container format e.g. FLV or WebM or rarely Mp4.

                                                                                                              For low-scale users, similar apps can be developed using WebRTC peer-to-peer methods, however such solutions lacks longer recordings (inside the app or browser) and lacks to connect more than 8-14 users (for single stream).

                                                                                                                    Muaz Khan Owner Fabio Neves • 4 years ago

                                                                                                                    You should try RTCMultiConnection-v3.0 and it will provide same features: https://github.com/muaz-khan/RTCMultiConnection/tree/master/RTCMultiConnection-v3.0

                                                                                                                    You simply need to make sure that "guests" CAN_NEVER_CALL "connection.open" method.

                                                                                                                    PS. RTCMultiConnection-v3.0 requires socket.io i.e. nodejs hosting.
